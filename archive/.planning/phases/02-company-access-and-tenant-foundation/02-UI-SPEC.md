## Source Of Truth

`xai-design.html` is the durable visual source of truth for this phase. It is a tracked design reference for XAI Books and must not be deleted, replaced, or treated as throwaway generated output. Phase 2 implementation must preserve its login split, authenticated shell, company switcher, company settings, activity/audit table, token palette, density, and owner-first copy tone.

Primary upstream sources used: `.planning/phases/02-company-access-and-tenant-foundation/02-CONTEXT.md`, `.planning/phases/01-monorepo-and-api-web-foundations/01-CONTEXT.md`, `.planning/REQUIREMENTS.md`, `frontend/src/styles.css`, `frontend/tailwind.config.ts`, and rendered inspection notes from `xai-design.html`.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, compact badges, inline metadata |
| sm | 8px | Menu item gaps, field label gaps, button icon gaps |
| md | 16px | Default form row spacing, card internals, table horizontal padding |
| lg | 24px | Drawer body padding, page head separation, sidebar section gaps |
| xl | 32px | Form column gaps, login form vertical grouping |
| 2xl | 48px | Login art panel padding, desktop page gutters when space allows |
| 3xl | 64px | Major authenticated-shell content breaks only |

Exceptions: icon-only and mobile tap targets must be at least 44px; desktop sidebar width is 264px; mobile sidebar width is 280px; desktop topbar height is 64px; mobile topbar height is 56px. Radius contract follows `xai-design.html`: 12px default, 8px small controls/menus, 6px inner menu hover rows. Existing 24px rounded primitives should be normalized down where they conflict with this reference.

## Typography

Use exactly these four display sizes and only weights 400 and 600 in Phase 2 UI. Financial and audit identifiers use tabular numerals.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 13.5px | 400 | 1.5 |
| Label | 12.5px | 600 | 1.2 |
| Heading | 20px | 600 | 1.2 |
| Display | 26px | 600 | 1.15 |

Secondary metadata, table timestamps, TRN chips, helper text, and compact badges must use the 12.5px Label size. Do not use viewport-scaled type. Letter spacing is 0 except uppercase labels/chips, which may use `0.06em`.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#f6f8fa` | App background, signed-out form background, empty-state background |
| Secondary (30%) | `#ffffff` / `#fbfcfd` | Cards, sidebar, topbar, dropdowns, tables, form panels |
| Accent (10%) | `#0ea5e9` | Primary CTA, active company/nav state, focus rings, selected company check, key status icon surfaces |
| Destructive | `#dc2626` | Remove member, revoke invite, destructive confirmations only |

Accent reserved for: primary submit buttons (`Sign in`, `Invite member`, `Save changes`), active nav item, active company context, focused form controls, selected company checkmark, loading progress on company switch, and audit event icon tones for safe/system events. Do not use cyan for every link or every hover state.

Supporting semantic colors: success `#16a34a` for completed invites/successful logins, warning `#d97706` for pending invites/session warnings, destructive ink `#b91c1c` for confirmed harmful actions. Borders use `#e7eaef` and `#eef1f4`; primary ink uses `#0b1220`, secondary ink `#3a475a`, muted ink `#6b7888`, faint ink `#97a2b0`.

## Phase 2 Interaction Contract

### Signed-Out And Auth States

Use the `xai-design.html` 50/50 login screen on desktop: dark brand/value panel left, light Clerk-compatible form panel right. On screens below 860px, hide the art panel and center the form in the light surface.

Clerk fields: work email, password, keep me signed in, forgot password, primary `Sign in`, secondary `Create a company workspace`. Footer text: `English (UAE) - Arabic coming soon`. Do not implement full Arabic or RTL UI in this phase.

Signed-in route guard: show authenticated app shell only inside Clerk `<SignedIn>`. Signed-out protected route redirects to the login state, not to a blank workspace. Session expired uses an alert panel above the form and keeps entered email when Clerk supports it.

### Authenticated App Shell

Use the `xai-design.html` shell: 264px left sidebar, white topbar, light gray content background, constrained content area, compact operational cards. Company context must be visible at all times through the app header company switcher; the sidebar remains navigation-only. Echo TRN metadata in the header when available.

Sidebar nav groups for Phase 2:

| Group | Items |
|-------|-------|
| Workspace | Dashboard placeholder, Company settings, Team & roles, Activity & audit |
| Compliance | VAT settings placeholder, Audit log |
| Finance | Disabled placeholders only; no finance feature screens in Phase 2 |

Do not make marketing hero sections inside the authenticated app. Cards are for repeated records, forms, metrics, tables, and empty states only. Do not nest cards inside cards.

### Company Switcher

Company switcher is the central Phase 2 interaction. It appears as a dropdown in the app header, per Phase 2 Decision D-17, using the `xai-design.html` compact menu styling for trigger, rows, spacing, and selected state:

| Element | Contract |
|---------|----------|
| Trigger | Company avatar initials, company name, business activity, chevron |
| Menu label | Switch company |
| Company row | Avatar initials, company name, business activity, selected checkmark |
| Add action | Add a company |
| Loading | Disable menu, show `Switching company...`, keep old company visible until `setActive` succeeds |
| Success | Clear TanStack Query cache and show `Company switched. Showing records for {company_name}.` |
| Failure | Keep old company active and show `Could not switch company. Try again or contact an admin.` |

Never render cached company-scoped data from the previous company after a switch. During the switch, replace scoped content with a compact loading state rather than preserving stale tables.

### Team, Invitations, And Roles

Team management uses a dense table on desktop and stacked rows on mobile. Fixed roles are `Owner`, `Admin`, `Accountant`, and `Viewer`; do not expose configurable role creation in Phase 2.

| Surface | Contract |
|---------|----------|
| Page title | Team & roles |
| Page description | Invite people, assign access, and keep this company private. |
| Primary CTA | Invite member |
| Invite form | Email, role select, optional message; submit in right-side drawer or modal |
| Role select | Use select menu with short role descriptions; default `Viewer` unless upstream data chooses otherwise |
| Member table columns | Name/email, Role, Status, Last active, Actions |
| Status badges | Active, Invite pending, Disabled |
| Permission hints | Show plain-language summaries, not permission IDs |

Role helper copy:

| Role | Helper copy |
|------|-------------|
| Owner | Full company access, including billing and company management. |
| Admin | Full company access except billing ownership. |
| Accountant | Can work with finance records, reports, and audit history. |
| Viewer | Can view records and reports but cannot make changes. |

### Permission Denied And 403 States

Permission failures must be visible, calm, and company-scoped. Do not silently hide the whole route when the backend returns 403.

| State | Contract |
|-------|----------|
| Route denied | White bordered panel on app background with lock icon, heading `You do not have access to this company`, body from copy contract, actions `Switch company` and `Back to workspace` |
| Action denied | Inline alert above affected table/form: `You do not have permission to do that in {company_name}.` |
| Cross-company 403 | Same route denied state; include attempted company name only if backend safely returns it |
| No companies | Signed-in onboarding panel: `Create a company workspace` as primary CTA |

Destructive or denied states use red only when access was removed or the user attempted a destructive action. Ordinary permission denials use neutral surface plus lock icon, not alarming red banners.

### UAE Company Settings

Use the `xai-design.html` company settings screen and keep scope to Phase 2. Logo upload, broad theming, and document presentation controls are placeholders/readiness only.

| Field | Contract |
|-------|----------|
| Legal name | Required text input |
| Business activity | Required text input or select if backend enum exists |
| TRN | Optional text input with hint `UAE Tax Registration Number` |
| VAT status | Select: VAT registered, Not VAT registered |
| VAT default | Read-only or select defaulted to `5% UAE VAT`; editable only if backend supports it |
| Default currency | Read-only `AED - UAE Dirham` |
| Registered address | Textarea |
| Interface language | Read-only `English (UAE)` |
| Arabic/RTL | Badge `Coming soon` |

Primary CTA: `Save changes`. Secondary CTA: `Discard changes`. Save success writes audit event and toast. Validation errors appear under fields with concise messages. Do not include logo upload implementation or brand color theming in Phase 2.

### Activity And Audit Log

Use the `xai-design.html` audit table pattern: white table container, thin borders, dense rows, icon/status circle, company-scoped copy.

| Element | Contract |
|---------|----------|
| Page title | Activity & audit log |
| Page description | Every important action - who did it, when, and what changed. Company-scoped. |
| CTA | Export log |
| Columns | Icon/status, Event, Actor, Details, When |
| Phase 2 events | Login, invite sent/revoked, role assignment, company settings update |
| Row density | 16px vertical cell padding, 13.5px body text |
| Time format | English (UAE-ready), absolute timestamp plus relative helper if available |

Export may be visible as a secondary/default button even if backend export is deferred, but it must be disabled with tooltip `Export will be available once audit export is connected` unless implemented.

## Responsiveness And Accessibility

Desktop: preserve the sidebar plus topbar shell. Tablet: keep sidebar if width supports it; otherwise use off-canvas navigation. Mobile below 760px: sidebar becomes a sheet/drawer, topbar hides search/TRN chip, page actions stack full width, tables scroll horizontally or render stacked record rows.

Keyboard and accessibility requirements:

| Area | Contract |
|------|----------|
| Focus | 3px cyan-soft focus ring on controls and menu items |
| Menus | Escape closes, arrow navigation supported by shadcn/Radix |
| Dialogs/drawers | Focus trapped, labeled title and description |
| Icons | Decorative icons `aria-hidden`; icon-only buttons require labels/tooltips |
| Errors | Field errors use `aria-describedby`; route errors use alert semantics |
| Contrast | Text and controls must meet WCAG AA on specified surfaces |
