# Feature Research

**Domain:** UAE-first cloud accounting and finance SaaS for SMEs
**Researched:** 2026-05-31
**Confidence:** HIGH

## Feature Landscape

### Table Stakes (Users Expect These)

Features users assume exist. Missing these = product feels incomplete.

| Feature | Why Expected | Complexity | Notes |
|---------|--------------|------------|-------|
| Secure login and user management | Cloud finance apps hold sensitive company data. | MEDIUM | Include invitations, user status, password reset, session handling, and basic account security. |
| Multi-company support | SME owners, groups, and accountants may manage more than one company. | HIGH | Tenant isolation must be designed into every table, query, role, file, and audit event. |
| Customers and suppliers | Needed before invoices, bills, receivables, and payables make sense. | LOW | Include TRN, billing address, contact people, payment terms, opening balances later if needed. |
| Sales invoices with UAE VAT | Core daily finance workflow and UAE-first promise. | HIGH | Store structured line items, VAT treatment, company TRN, customer details, and invoice status. |
| Printable PDF invoices | SMEs need customer-facing documents with branding and VAT details. | MEDIUM | PDF is an output format, not the source of truth. |
| Payments and receivables tracking | Owners need to know who paid and who still owes money. | MEDIUM | Support full, partial, and over/under-payment handling eventually. |
| Expenses and supplier bills | Daily finance loop is incomplete without spend capture. | MEDIUM | Link supplier, VAT treatment, due date, attachments, status, and payment state. |
| Approval workflows | SMEs need control over invoices/bills before posting or sending. | MEDIUM | Start constrained: draft -> pending approval -> approved -> sent/posted. |
| Chart of accounts | Required for real accounting and reporting. | HIGH | Provide UAE SME defaults while allowing controlled customization. |
| Double-entry bookkeeping | Financial statements and VAT summaries depend on reliable postings. | HIGH | Journal entries should be generated by business events, not manually patched. |
| VAT summary report | UAE businesses need visibility into VAT collected and paid. | MEDIUM | Include sales VAT, purchase VAT, net payable/reclaimable, and supporting drill-downs. |
| Dashboard | Owners expect immediate visibility into cash, receivables, payables, revenue, expenses, and VAT position. | MEDIUM | Keep owner-first and plain-language. |
| Audit/activity log | Trustworthy principle requires who/what/when/changed history. | HIGH | Append-only events with before/after diffs for sensitive changes. |

### Differentiators (Competitive Advantage)

| Feature | Value Proposition | Complexity | Notes |
|---------|-------------------|------------|-------|
| UAE-first defaults | Product feels native rather than adapted from a generic global system. | MEDIUM | AED, TRN, 5% VAT, UAE document expectations, VAT summary, bilingual-ready labels. |
| Owner-first UX over accounting backbone | Lets non-accountants run daily finance while accountants still trust the books. | HIGH | Requires careful language, guided workflows, and hidden-but-correct ledger behavior. |
| Configurable fields and templates | SMEs can adapt the system to their workflow without custom development. | MEDIUM | Use typed custom fields with validation and scoped availability. |
| Admin-level roles, permissions, workflow rules | Supports growth from owner-only to team operation. | HIGH | Keep v1 permission model understandable and testable. |
| Future eInvoicing readiness | Positions XAI for regulatory evolution. | HIGH | Store structured invoice data and provider-neutral statuses now, integrate later. |
| Full audit trail as product feature | Helps owners, finance staff, and accountants resolve disputes and changes. | MEDIUM | Make activity visible in context, not buried in admin tools only. |

### Anti-Features (Commonly Requested, Often Problematic)

| Feature | Why Requested | Why Problematic | Alternative |
|---------|---------------|-----------------|-------------|
| Full ERP scope in v1 | Feels like "complete business management." | Dilutes accounting launch and slows validation. | Build XAI Books first, with platform extension points. |
| Full Arabic UI in v1 | UAE-first implies Arabic support. | Translation/RTL quality is expensive and easy to do poorly. | Build bilingual-ready architecture and add full Arabic later. |
| Highly generic workflow builder | Configurability sounds powerful. | Hard to test, confusing for SMEs, and risky for accounting controls. | Constrained workflows with configurable statuses, approvals, roles, fields, and templates. |
| Manual journal-first UX | Accountants may ask for it. | Owner-first users should not need accounting jargon for daily work. | Generate ledger entries from invoices, payments, expenses, and approvals; add controlled manual journals later. |
| PDF invoice as the canonical invoice | Easy to demo and print. | Blocks eInvoicing readiness and creates data extraction problems. | Structured invoice model with PDF rendering. |

## Feature Dependencies

```text
Company + Users + Roles
    -> Multi-company isolation
        -> Customers/Suppliers
            -> Invoices + Expenses/Bills
                -> VAT calculations
                -> Approvals
                -> Payments
                -> Ledger postings
                    -> VAT summary + Dashboard + Reports
                        -> Audit review and evidence

Structured invoice data
    -> PDF invoices
    -> Future eInvoicing integration

Custom fields + templates
    -> Configurable invoices/contacts/expenses
    -> Workflow rules
```

### Dependency Notes

- **Multi-company isolation precedes everything:** every record and file must belong to a company before business workflows are safe.
- **Chart of accounts precedes meaningful reports:** invoices and bills can be captured early, but financial statements require correct posting rules.
- **Structured document data precedes PDFs and eInvoicing:** render from data; do not parse PDFs later.
- **Permissions and audit should be foundational:** approval and trust features are weak if added after workflow data already exists.

## MVP Definition

### Launch With (v1)

Minimum viable product needed to validate XAI Books as an owner-first UAE finance system.

- [ ] Secure login, users, roles, and company membership.
- [ ] Multi-company data isolation.
- [ ] Company settings with AED, TRN, VAT registration flag, and 5% UAE VAT default.
- [ ] Customer and supplier records.
- [ ] Sales invoices with line items, VAT calculation, status, approval, PDF output, and payment tracking.
- [ ] Expenses and supplier bills with VAT treatment, status, attachments, and payment tracking.
- [ ] Chart of accounts and event-driven double-entry postings for core workflows.
- [ ] VAT summary report with supporting drill-downs.
- [ ] Owner dashboard for cash, receivables, payables, sales, expenses, and VAT position.
- [ ] Activity/audit log for sensitive actions and record changes.
- [ ] Configurable custom fields and templates for key records.

### Add After Validation (v1.x)

- [ ] Bank import/reconciliation - add when users need faster transaction matching.
- [ ] Recurring invoices/expenses - add after base invoice and expense flows are stable.
- [ ] Accountant review tools - add once SMEs invite external accountants.
- [ ] More detailed financial statements - add after ledger correctness is validated.
- [ ] Arabic document output - add after bilingual-ready data/copy structure is in place.

### Future Consideration (v2+)

- [ ] Full Arabic/RTL interface - important for UAE expansion but not Phase 1.
- [ ] Inventory - common in accounting tools, but not essential for daily finance validation.
- [ ] Payroll - UAE-specific complexity; defer.
- [ ] Corporate tax return workflow - important later, but VAT/daily bookkeeping comes first.
- [ ] UAE eInvoicing ASP integration - prepare data model now; integrate when scope and customer segment demand it.
- [ ] Multi-currency - useful for import/export SMEs but not needed for AED-first v1.
